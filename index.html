1) Top-level 옵션
변수 타입 현재 값 설명
pauseWhenHidden boolean true 브라우저 탭 비활성화 시 자동 일시정지하고, 다시 활성화되면 자동 재개합니다.
reducedMotionBehavior "instant" | "animate" "instant" prefers-reduced-motion 사용자에게 애니메이션 대신 즉시 완성 상태를 보여줍니다.
reveal.selector string ".hero-reveal" 후속 노출(리빌) 대상을 선택하는 CSS 선택자입니다.
reveal.baseDelay number(ms) 90 리빌 시작 지연 시간입니다.
reveal.step number(ms) 160 대상 간 리빌 간격입니다. data-hero-order와 함께 순차 노출됩니다.
startTrigger "immediate" | "in-view" | "interaction" 프리셋별 상이 애니메이션 시작 조건입니다. 즉시/뷰포트 진입/사용자 이벤트 기반 시작을 선택합니다.
startTriggerTarget / startTriggerEvent string / string "#hero-stage" 또는 "#hero-typed" / "click" 시작 트리거 대상과 이벤트 타입입니다. 클릭 시작/특정 영역 진입 시작에 사용합니다.
replay.mode / replay.cooldownMs "once" | "manual" | "on-visible" / number 프리셋별 상이 재생 정책과 최소 재생 간격입니다. 스크롤 재진입 리플레이 제어에 사용합니다.
a11y.skipEnabled / a11y.skipKey / a11y.ariaLive boolean / string / string true / "Escape" / "polite" 키보드 Skip과 스크린리더 알림 강도를 설정합니다.
interaction.* object pauseOnHover, clickToSkip, clickToReplay 호버 일시정지/클릭 스킵/클릭 리플레이 등 상호작용 기반 제어를 제공합니다.
pace.* object minDelay, maxDelay, punctuationPause ... 타이핑 리듬을 수치로 조정합니다. 테스트 재현성과 UX 톤 튜닝에 유용합니다.

2) lines[] 라인별 옵션
변수 타입 예시 설명
containerId string "hero-typed" 라인 컨테이너 요소 ID입니다. 이 요소의 data-text를 기본 텍스트로 사용합니다.
trackId string "hero-typed-track" 문자가 실제로 렌더링되는 내부 트랙 요소 ID입니다.
strategy "plain" | "ime-ko" | "instant" "ime-ko" plain: 일반 타이핑, ime-ko: 한글 조합 프레임 타이핑, instant: 즉시 출력
delayBefore number(ms) 260 해당 라인 시작 전 대기 시간입니다. 첫 라인 인트로에 주로 사용합니다.
delayAfter number(ms) 680 / 1000 해당 라인 종료 후 대기 시간입니다. 다음 라인 또는 리빌 시작 텀을 제어합니다.
activateClass string "is-active" 라인 시작 시 컨테이너에 부여할 클래스입니다. 2줄 fade-in/slide-in 트리거로 사용합니다.

3) 현재 실행 코드
const demoConfig = {
  pauseWhenHidden: true,
  reducedMotionBehavior: "instant",
  reveal: {
    selector: ".hero-reveal",
    baseDelay: 90,
    step: 160
  },
  lines: [
    {
      containerId: "hero-typed",
      trackId: "hero-typed-track",
      strategy: "plain",
      delayBefore: 260,
      delayAfter: 680
    },
    {
      containerId: "hero-typed-second",
      trackId: "hero-typed-second-track",
      strategy: "ime-ko",
      activateClass: "is-active",
      delayAfter: 1000
    }
  ]
};

window.demoTypingPlayer = window.TypingAnimationLib.create(demoConfig);

4) 다양한 옵션 형태 예시

선택자 기반 지정

container: "#hero-typed", track: "#hero-typed-track"처럼 ID 대신 selector 문자열을 그대로 전달.

텍스트 직접 주입

text를 주면 data-text보다 우선 적용. API/상태값 기반 문구 동적 반영에 적합.

속도 랜덤 고정

randomFn: () => 0.15로 타이핑 지연 랜덤성을 고정해 테스트 재현성 확보.

수동 시작 모드

autoStart: false 후 player.play() 호출로 인터랙션 트리거 시작.

뷰포트 시작 + 재진입 리플레이

startTrigger: "in-view", replay.mode: "on-visible" 조합으로 스크롤 맥락형 재생.

인터랙션 제어

interaction.pauseOnHover, clickToSkip, clickToReplay로 사용자 주도 제어.

접근성 제어

a11y.skipEnabled, a11y.skipKey, a11y.ariaLive로 키보드/스크린리더 대응.

리듬 튜닝

pace.minDelay/maxDelay/punctuationPause로 타이핑 템포를 브랜드 톤에 맞춰 조정.

이벤트 훅

hooks.onLineStep, hooks.onComplete로 진행률/로깅/분석 이벤트를 연결.
